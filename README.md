# Homework-2

## Business Workflow
This project focuses on summarizing meetings into clear action items.

## Who the User Is
The primary user is a team member, project manager, or meeting organizer who needs a fast and reliable summary after a meeting.

## Input
The system receives meeting notes, a meeting transcript, or a rough written summary of what was discussed.

## Output
The system produces a structured summary of action items, including:
- task description
- owner if mentioned
- deadline if mentioned
- follow-up notes or unresolved questions

## Why This Task Is Valuable
Meetings often generate useful decisions, but the next steps can become unclear or forgotten. A system that turns meeting content into action items can save time, improve accountability, and make follow-up easier. However, human review is still important when ownership, deadlines, or decisions are ambiguous.
